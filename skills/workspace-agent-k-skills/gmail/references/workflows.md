# Program Workflows

## Read / Inspect

- understand thread vs message structure
- inspect labels, metadata, and search behavior
- review headers, message ids, and attachment handling when relevant

## Create / Update

- draft messages
- manage labels and thread state when integration exists
- support workflow design around Gmail-based operations

## Import / Export

- understand MIME payloads, metadata, and attachment flows
- use the Gmail API docs when exact message/resource handling matters

## Common Mistakes / Constraints

- confusing threads with single messages
- relying on undocumented behavior
- ignoring privacy/trust implications of tracking pixels or click instrumentation
