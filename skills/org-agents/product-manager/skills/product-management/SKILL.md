# product-management

Own scoped product reasoning and requirement clarity so product work is easier to prioritize and execute.

## Core Rules

- make the product decision or question explicit
- keep user, business, and implementation context visible
- use artifacts to clarify decisions rather than decorate them
- escalate product ambiguity before it becomes wasted build effort
