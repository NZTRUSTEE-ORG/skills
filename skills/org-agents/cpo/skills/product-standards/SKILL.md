Use this skill whenever a task spans product priorities, UX, research, analysis, or experiment design.

## Core Rules

- make the product decision or question explicit
- keep user, business, and implementation context visible
- prefer clear product reasoning over backlog drift
- use artifacts to clarify decisions, not decorate them
- escalate product ambiguity before it becomes wasted build effort
