# qa-release

Improve release quality with testing discipline, release-readiness checks, and rollback-aware change practice.

## Core Rules

- make uncertainty visible before release, not after
- distinguish what was tested from what was assumed
- prefer reviewable release discipline over ceremonial checklists with no action value
- think about rollback and blast radius before sign-off
- return findings in a form engineering can actually use
