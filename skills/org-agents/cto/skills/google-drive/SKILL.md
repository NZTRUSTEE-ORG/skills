# Google Drive Skill

## Purpose

Work effectively with Google Drive in CTO-context workflows.

## Use This Skill For

- locating, organizing, and reviewing Drive-hosted technical material
- handling architecture docs, technical plans, specs, runbooks, exports, and supporting files
- preparing artifacts for engineering leadership review or cross-functional sharing
- using sanctioned Google Workspace wrappers when live Drive operations are needed
- producing files intended for CTO role folders and governed Drive destinations

## Read These References As Needed

- `references/overview.md` for what the program is used for
- `references/workflows.md` for common tasks
- `references/docs-links.md` for official documentation
- `references/files-and-formats.md` for supported file types and handling guidance

## Rules

- prefer official docs when exact behavior matters
- treat the org’s sanctioned Google Workspace wrapper path as the default operational route for live Drive work
- distinguish conceptual familiarity from tool-enabled operations
- preserve folder, permission, and governance expectations when moving or creating technical artifacts
- default CTO-local outputs to `agents/cto/My Agent Documents` and Drive outputs to `EDIT ACCESS/Agents/cto/My Agent Documents` unless the task specifies another destination
- if no live integration is available in the current context, fall back to file-based or documentation-based work without claiming Drive is categorically unavailable

## Operating Mode

This agent is expected to operate Google Drive actively for CTO work when sanctioned tooling, wrappers, or files are available, and otherwise work from exports, local artifacts, and documentation with the same governance constraints.
