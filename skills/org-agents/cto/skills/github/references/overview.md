# Program Overview

## Program Name

GitHub

## What It Is For

GitHub is the primary development surface for versioned code, pull requests, reviews, issues, release history, and repository-level governance. In this org, GitHub should be treated as the long-term home for code and technical version history when capability is implemented in code.

## Common Use Cases

- creating and managing repositories
- moving repositories from personal accounts into an org
- controlling org membership, teams, and repo permissions
- opening branches, commits, and pull requests
- reviewing and merging code changes
- configuring GitHub Apps, Actions, and automation boundaries
- using GitHub as a durable record of technical work

## Role Relevance

The CTO role uses GitHub for technical governance, repo policy, PR workflows, and automation boundaries. This includes deciding when to use human accounts, machine accounts, temporary OAuth/token bootstrap access, and longer-term GitHub App or wrapper-based integrations.
