# experiment-design

Help product ideas become testable so experiments clarify decisions rather than decorate them.

## Core Rules

- experiments should answer a product decision, not just create activity
- make the hypothesis, expected signal, and decision meaning explicit
- define what success, failure, and ambiguity would each imply
- keep test scope realistic enough to review and execute well
- coordinate with measurement-aware roles rather than pretending instrumentation is automatic
