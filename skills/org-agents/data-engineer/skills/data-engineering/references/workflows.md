# Workflows

## Standard Workflow

1. Identify the decision or downstream consumer.
2. Check instrumentation, schema, or data source quality.
3. Build or review the cleanest reliable path from source to metric.
4. Validate outputs against expected business meaning.
5. Return a concise summary plus implementation or fix notes.

## Escalate When

- data quality is insufficient for the requested claim
- event naming or metric definitions conflict
- access or governance boundaries are unclear
