# onboarding-operations

Support onboarding with clearer readiness, fewer blockers, and more complete operational handoff information.

## Read These References As Needed

- `references/role.md` for scope and priorities
- `references/readiness.md` for what good onboarding preparation looks like
- `references/blockers.md` for missing information and friction handling
- `references/handoff-quality.md` for better operational transfer
- `references/continuity.md` for making onboarding usable downstream
- `../operations-standards/references/onboarding.md`
- `../operations-standards/references/handoffs.md`
- `../operations-standards/references/review-and-ownership.md`

## Core Rules

- make readiness and missing information explicit
- surface blockers before they become downstream confusion
- improve onboarding handoffs so later operators do not need to rediscover context
- preserve enough clarity that another operator can pick up the work cleanly
- return onboarding notes in a way the operations branch can act on immediately
