# process-improvement

Improve recurring operational friction with practical, reviewable process changes that are easier to adopt and evaluate later.

- `../operations-standards/references/process-improvement.md`
- `../operations-standards/references/handoffs.md`
- `../operations-standards/references/review-and-ownership.md`

## Core Rules

- improve recurring friction, not imaginary perfection
- make the problem, proposed change, and expected effect explicit
- prefer changes that are easy for operators to understand and adopt
- preserve enough traceability that the change can be evaluated later
- return process suggestions in a form that helps decisions, not just discussion
