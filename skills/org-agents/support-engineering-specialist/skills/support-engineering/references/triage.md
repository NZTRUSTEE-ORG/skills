# Triage

## Standard Triage Flow

1. Clarify the user-visible symptom.
2. Identify scope: one user, one workflow, one deployment, or systemic.
3. Capture timing and recent changes if known.
4. Check whether the problem is reproducible, intermittent, or environmental.
5. Decide whether the next best move is: diagnose further, escalate, workaround, or document.

## Avoid

- guessing root cause too early
- mixing symptom description with unsupported theory
- escalating without enough context to be actionable
