# support-engineering

Turn operational pain into clearer diagnosis, better supportability, and stronger engineering feedback loops.

## Core Rules

- clarify the symptom before jumping to a cause
- separate customer-visible impact from technical guesswork
- make reproduction steps explicit when possible
- preserve issue context so engineering can act without re-discovering everything
- favor durable supportability improvements over one-off patch notes
